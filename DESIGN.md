# Number Chain — 设计文档

## 一、游戏规则

**目标**：在网格中用一条连续路径，按升序或降序将所有非封锁格填满（路径从 1 到 N，N = 非封锁格总数）。

**路径规则**
- 支持 8 方向移动（水平、垂直、对角线）
- 每个格子只能被经过一次（一进一出）
- 两条路径的线段可在格子之间形成 X 形视觉交叉，但格子本身不共用
- 从固定数字格出发，自动按升序/降序填写经过的空白格

**完成判定**：进度达到 100%（所有非封锁格都被覆盖）

---

## 二、谜题格式（CSV）

```
行数,列数
格1,格2,...
...
```

| 值 | 含义 |
|----|------|
| 数字（如 `1`、`7`、`22`） | 固定提示格，值已确定 |
| `0` | 空白可填格 |
| `X` | 封锁格，不可通过 |

**示例（5×5，共 22 格，1→22）**
```
5,5
1,0,0,X,15
0,0,0,0,0
7,X,0,0,0
0,0,0,X,0
0,0,22,0,0
```

## 三、节点样式

| 状态 | 外观 |
|------|------|
| 空白格 | 暗色细边框圆环，无数字 |
| 固定格 | 金黄 `#e8c468`，粗边框，显示数字 |
| 封锁格 | 深灰 `#222`，不可点击 |
| 已填格（普通路径） | 深蓝 `#1e3a6e`，蓝色数字 |
| 已填格（唯一路径） | 深绿 `#0d3a22`，绿色数字 |
| 当前拖拽中 | 亮蓝，轻微放大效果 |

**连接线**：从圆边缘到圆边缘（不穿过圆心），普通=蓝色，唯一=绿色，拖拽中=淡蓝

---

## 四、操作方式

| 操作 | 效果 |
|------|------|
| 从固定格拖拽 | 新建路径段，沿途格自动编号 |
| 从路径端点拖拽 | 延伸该路径 |
| 从路径中间格拖拽 | 截断该格之后的路径，从此处重新延伸 |
| 回拖（向前一格移动） | 撤销最后一步 |
| 升序 / 降序切换 | 决定新建路径时数字递增还是递减方向 |
| 唯一路径开关 | 开启后画的线变为绿色（标记已确认的路径） |
| 擦除模式开关 | 开启后点击格子：端点→直接删除，中间→拆分成两段 |
| 重置全部 | 清空所有路径，保留谜题 |

## 五、技术架构

**单文件**：`index.html`（HTML + CSS + JS，无依赖，浏览器直接打开）

**布局**
- `#game-area`：相对定位容器
  - `#svg-overlay`：SVG，绝对定位在底层，绘制连线
  - `#cells-layer`：CSS Grid，绝对定位在顶层，绘制圆形节点

**事件**：Pointer Events API（`pointerdown` + `setPointerCapture` + `pointermove` + `pointerup`），确保拖拽不丢失、文字不被选中

**核心数据结构**
```js
puzzle = { rows, cols, grid: [[{type, value}]] }

chains = [{
  cells: [[r, c], ...],  // 有序格子坐标
  firstVal: number,       // cells[0] 的数值
  ascending: bool,        // true=升序
  unique: bool            // true=绿色唯一路径
}]

active = {  // 正在拖拽中的路径（同 chains 结构 + chainIdx/fromStart）
  cells, firstVal, ascending, unique,
  chainIdx,   // -1=新建，>=0=延伸已有链
  fromStart   // true=从链头延伸（向前插入）
}
```

**值计算**：`cellValue(chain, index) = chain.firstVal + (ascending ? index : -index)`

**格子尺寸自适应**：根据 `Math.max(rows, cols)` 动态计算 `CELL`（直径）和 `GAP`（间距），最大区域 520px

## 六、已知限制

- 谜题不自动生成，由用户提供 CSV
- 不持久化存档（刷新即重置）
- 不校验谜题是否有唯一解
- 不支持连接两条独立链的端点（未实现合并逻辑）
